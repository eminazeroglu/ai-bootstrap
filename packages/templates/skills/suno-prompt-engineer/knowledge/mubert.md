# Mubert — Model Specification

> Real-time generative music + royalty-free music API. Built for streaming, content creation, apps.

## Model identity
- **Provider**: Mubert (mubert.com)
- **Generates**: Streaming music + downloadable tracks
- **Output format**: MP3 / WAV / Streaming
- **Cost basis**: Subscription tiers + API access
- **Specialty**: Royalty-free music for content, real-time streaming
- **Founded**: 2016

## Strengths
- **Royalty-free** for commercial use (paid tier)
- **Real-time generation** (streaming for apps/games)
- **API access** (developers can integrate)
- Content creator focus (YouTube, TikTok, Twitch)
- Mood-based simple prompts
- Length flexibility (instant + endless)
- Low latency for streaming
- Wide style coverage (pop, electronic, ambient, hip-hop, etc.)

## Weaknesses
- Less "composer-grade" output (more functional than artistic)
- Vocals very limited
- Not for full-song lyric work
- Less control than Suno/Udio
- Output sometimes formulaic
- Multilingual not relevant (instrumental focus)

## Use cases
- **YouTube background music** (royalty-free)
- **TikTok / Reels content**
- **Game soundtracks** (real-time generation)
- **App background audio**
- **Live streaming** (Twitch, etc.)
- **Podcast underscoring**
- **Workout / focus / sleep apps**

## Workflow

### Mubert Render (most common)
1. Pick mood + genre
2. Set duration
3. Generate
4. Download royalty-free track

### Mubert API (developers)
1. Integrate API
2. Generate music in your app/game in real-time
3. Reactive to user behavior possible

### Mubert Studio
- More refined generation control
- Stem options (paid tier)

## Prompt parameters (Mubert Render)
- **Mood**: Energetic / Calm / Focused / Happy / Sad / etc.
- **Genre**: Pop / Hip-hop / Electronic / Ambient / etc.
- **Duration**: 10s to 25 min
- **Tempo**: Slow / Mid / Fast (or BPM)
- **Activity**: Workout / Sleep / Focus / Party

## Best uses
- **Content creator background music** (no copyright strikes)
- **Mobile app music** (real-time)
- **Game adaptive music** (mood-reactive)
- **Quick functional underscore** (no artistic claim)
- **Podcast music**
- **Commercial advertising** (paid tier)

## Royalty-free advantage
For YouTube/TikTok creators, Mubert solves the music licensing problem:
- No DMCA strikes
- No copyright claims
- No revenue split with rights holders
- Use forever once generated (paid tier)

## When to use Mubert vs others
- **Mubert**: royalty-free background, content creation, real-time apps
- **Suno**: artistic full songs with vocals
- **Udio**: hip-hop / electronic beats with quality
- **Stable Audio**: sound design + commercial loops
- **AIVA**: composer-grade orchestral with MIDI

## Tier comparison
- **Free**: limited downloads, attribution required
- **Creator**: YouTube/TikTok use, no attribution
- **Pro**: streaming services, advertising rights
- **Enterprise**: API access, real-time integration

## Best practices
- Think FUNCTIONAL, not artistic (it's not Suno)
- Use for backing track / underscore, not the song itself
- Pair with your own voiceover for content
- Use real-time API for adaptive game/app music
- Don't expect editable stems on lower tiers

## Anti-patterns
- Don't use for vocal song writing
- Don't expect Suno-level "song" output
- Don't compare to artistic AI music tools — different goal
- Don't use for art project — it's a content/utility tool
- Don't ignore the API for app/game development
